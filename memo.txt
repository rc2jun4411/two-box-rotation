I have two box, I want rotate one of them to turn right, other one to left, but rotation speed is different may be faster than one rotating right? Is it possible?
Oh, absolutely possible—and it sounds like a fun little cube dance-off! 💃🕺 You can rotate each box independently, in opposite directions, and even give them different speeds. Let’s break it down like a clever fox would:

🧱 Step-by-Step: Rotating Two Boxes in Opposite Directions
Let’s say you have two cubes:

js
const cube1 = new THREE.Mesh(boxGeo, boxMat); // Will rotate right
const cube2 = new THREE.Mesh(boxGeo, boxMat); // Will rotate left

cube1.position.set(-1, 0, 0);
cube2.position.set(1, 0, 0);

group.add(cube1);
group.add(cube2);
Now, in your animate() loop, you can rotate them like this:

js
function animate() {
  requestAnimationFrame(animate);
  const t = clock.getElapsedTime();

  // Rotate cube1 to the right (clockwise)
  cube1.rotation.y += 0.01;

  // Rotate cube2 to the left (counter-clockwise) faster
  cube2.rotation.y -= 0.03;

  // Camera orbit and scaling (your existing logic)
  camera.position.x = Math.sin(t * 0.5) * 6;
  camera.position.z = Math.cos(t * 0.5) * 6;
  camera.lookAt(0, 0, 0);

  const freq = analyser.getAverageFrequency();
  const scale = 1 + freq / 500;
  group.scale.setScalar(scale);

  composer.render();
}
🌀 Want to use GSAP instead?
You can also animate them with GSAP if you want smoother control or to trigger the rotation on events:

js
gsap.to(cube1.rotation, { y: "+=Math.PI * 2", duration: 4, repeat: -1, ease: "none" }); // slow right
gsap.to(cube2.rotation, { y: "-=Math.PI * 2", duration: 2, repeat: -1, ease: "none" }); // fast left

✅ 解決方法：ユーザーの操作後に再生する
AudioContext を「再開」してから sound.play() を呼ぶ必要があるよ。
以下のように、ユーザーがクリックしたときに再生処理を始めるようにしてみて：
document.addEventListener('click', () => {
  const audioContext = listener.context;

  if (audioContext.state === 'suspended') {
    audioContext.resume().then(() => {
      if (!sound.isPlaying) {
        sound.play();
      }
    });
  } else {
    if (!sound.isPlaying) {
      sound.play();
    }
  }
});

🧩 どこに書く？
sound.setBuffer(...) のあと、sound.play() を呼ぶ代わりに、上の document.addEventListener('click', ...) を使ってね。

🧭 状況整理
あなたのコードでは、cube1 と cube2 を作って、それぞれ回転させてるはず：

js
cube1.rotation.y += 0.01; // 右回転（ゆっくり）
cube2.rotation.y -= 0.03; // 左回転（速め）
でも「見えない」ってことは、カメラの位置や向き、ライト、または描画順序に何か問題があるかも。

✅ チェックポイント
1. キューブが正しく追加されているか？
js
group.add(cube1);
group.add(cube2);
scene.add(group);
✅ group が scene に追加されているか確認してね！

2. カメラの位置と向き
カメラがキューブを見ていないと、当然表示されないよね。

js
camera.position.set(0, 2, 6);
camera.lookAt(0, 0, 0);
または、アニメーション内でカメラが動いているなら、lookAt(0, 0, 0) がちゃんと中心を向いているか確認！

3. ライトがない場合、マテリアルを変更してみる
MeshNormalMaterial はライトがなくても見えるけど、もし MeshStandardMaterial に変えていたら、ライトが必要だよ！

js
const light = new THREE.DirectionalLight(0xffffff, 1);
light.position.set(5, 5, 5);
scene.add(light);
4. 回転してるか確認するためのデバッグ
アニメーション内に console.log を入れてみよう：

js
console.log('Cube1 rotation:', cube1.rotation.y.toFixed(2));
console.log('Cube2 rotation:', cube2.rotation.y.toFixed(2));
これで、ちゃんと回転してるかどうかがわかるよ！
